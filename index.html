Komenda Opis działania Akcja
cd Zmienia bieżący folder, w którym znajduje się użytkownik. Przykładowo cd C:\Skrypty przenosi do wskazanego katalogu.
cls Czyści całkowicie ekran konsoli ze wszystkich dotychczas wyświetlonych poleceń i komunikatów.
dir Wyświetla listę plików oraz podfolderów znajdujących się w obecnej lokalizacji użytkownika.
echo > plik.txt Tworzy nowy, pusty plik tekstowy o podanej nazwie (lub nadpisuje istniejący) w bieżącym katalogu.
ipconfig Wyświetla szczegółowe informacje o konfiguracji sieciowej wszystkich kart sieciowych (adres IP, maska podsieci, brama domyślna).
ping Testuje poprawność i opóźnienie komunikacji sieciowej z innym urządzeniem o podanym adresie IP lub domenie.
copy / xcopy Służy do kopiowania pojedynczych plików (copy) lub całych struktur folderów i drzew katalogów wraz z plikami (xcopy).
del Usuwa jeden lub więcej wskazanych plików z dysku.
mkdir Tworzy nowy katalog (folder) w bieżącej lub określonej ścieżce lokalizacji.
move Przenosi pliki lub całe foldery z jednej lokalizacji do drugiej (może służyć też do zmiany nazwy).
rename Zmienia nazwę wskazanego pliku lub grupy plików.
rmdir Usuwa pusty katalog z systemu plików.
tree Graficznie wyświetla strukturę folderów i plików w postaci czytelnego drzewa katalogów.
nslookup Narzędzie diagnostyczne służące do sprawdzania działania i odpytywania serwerów DNS w celu mapowania domen na adresy IP.
hostname Wyświetla unikalną sieciową nazwę bieżącego komputera.
ver Wyświetla dokładny numer i wersję posiadanego systemu operacyjnego Windows.
ftp Uruchamia wbudowanego klienta protokołu FTP, pozwalając na połączenie i przesyłanie plików z serwerem FTP.
ssh user@adres Uruchamia połączenie z szyfrowanym terminalem protokołu SSH w celu zdalnej administracji urządzeniem/serwerem.
Komenda Opis działania i parametry Akcja
sudo polecenie Uruchamia dane polecenie z uprawnieniami superużytkownika (root) - administratora systemu.
cd Zmienia bieżący katalog roboczy użytkownika w drzewie systemowym.
ls -la Wyświetla szczegółową listę wszystkich plików i folderów w obecnej lokalizacji, wliczając pliki ukryte (z kropką na początku).
clear Czyści widok i bufor konsoli terminala.
sudo fdisk dysk Narzędzie do tworzenia, usuwania i edycji tablicy partycji na wskazanym dysku twardym (np. /dev/sdb).
sudo mkfs -t system dysk Formatuje wskazaną partycję dysku do wybranego systemu plików (np. -t ext4).
sudo mount dysk folder Montuje partycję dyskową pod określony w strukturze systemowej katalog (punkt montowania).
touch plik.txt Tworzy nowy, pusty plik lub aktualizuje datę modyfikacji pliku jeśli już istnieje.
sudo nano plik.txt Otwiera plik w prostym, wbudowanym tekstowym edytorze konsolowym nano z prawami administratora.
ip a Wyświetla pełne informacje o konfiguracji interfejsów sieciowych, adresach IP (IPv4/IPv6) oraz statusie połączenia.
sudo chmod uprawnienia Zmienia uprawnienia dostępu (odczyt, zapis, wykonanie) do pliku lub folderu (np. chmod 755 plik).
sudo chown właściciel:grupa Ustawia lub zmienia właściciela oraz grupę powiązaną ze wskazanym plikiem bądź katalogiem.
sudo apt install pakiet Pobiera i instaluje wybrany pakiet/oprogramowanie z oficjalnych repozytoriów dystrybucji.
sudo systemctl [akcja] usługa Zarządza usługami systemowymi. Kluczowe akcje:
  • status - sprawdza aktualny stan działania usługi
  • restart - restartuje usługę (konieczne po edycji konfiguracji)
  • start / stop - uruchamia lub zatrzymuje proces usługi
  • enable - dodaje usługę do autostartu systemu
sudo adduser nazwa_usera Interaktywnie dodaje nowego użytkownika do systemu wraz z katalogiem domowym.
  • --home lokalizacja: wymusza inną lokalizację folderu domowego
  • --uid id: nadaje konkretny identyfikator użytkownika
  • --disabled-login: tworzy konto bez możliwości logowania
  • --shell powłoka: ustawia inną powłokę (np. /bin/bash)
sudo usermod nazwa_usera Modyfikuje istniejące ustawienia konta użytkownika.
  • -G grupa: przypisuje użytkownika do dodatkowej grupy
  • -e data: ustawia datę wygaśnięcia ważności konta (RRRR-MM-DD)
  • -d lokalizacja: zmienia ścieżkę katalogu domowego
  • -s powłoka: zmienia domyślną powłokę użytkownika
  • -l nowa_nazwa: zmienia login (nazwę użytkownika)
  • -u nowe_uid: modyfikuje identyfikator UID
sudo chage nazwa_usera Zarządza polityką wygasania i ważności haseł użytkowników.
  • -l: wyświetla szczegółowe reguły ważności hasła danego konta
  • -m liczba: minimalna liczba dni dopuszczalna między zmianami hasła
  • -M liczba: maksymalna liczba dni ważności hasła użytkownika

💡 Szybkie Notatki Diagnostyczne

  • Sprawdzenie sieci (Windows): Użyj ipconfig /all, aby zobaczyć serwery DHCP i DNS.
  • Sprawdzenie sieci (Linux): Użyj ip a lub nmcli.
  • Test serwera DNS: Wywołaj nslookup nazwa_domeny. Pozwala sprawdzić, czy serwer prawidłowo tłumaczy nazwy na adresy IP.
  • Test komunikacji: ping adres_IP sprawdza warstwę trzecią modelu OSI (sieciową).

⚙️ Kluczowe Usługi Egzaminacyjne

  • DHCP: Odpowiada za automatyczne przydzielanie adresów IP w sieci. Na egzaminie pamiętaj o konfiguracji zakresu wykluczeń i zastrzeżeń (rezerwacji) adresów MAC.
  • Active Directory (AD DS): Baza danych i usługa katalogowa Microsoftu. Promowanie do kontrolera wykonasz przez Menedżer Serwera. Pamiętaj o dodawaniu jednostek organizacyjnych (OU) i użytkowników.
  • Netplan (Ubuntu): Pliki konfiguracyjne znajdują się w /etc/netplan/. Pamiętaj o poprawnej strukturze wcięć (używaj tylko spacji, nie tabulatorów) i zatwierdzeniu zmian przez sudo netplan apply.
